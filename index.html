Learn Git With Me: Step-by-Step Git Workflow

This guide will help you understand Git commands from the basics to linking your local Git repository with a GitHub remote repository.

Step-by-Step Guide to Git Commands

1. Install Git

First, ensure Git is installed on your system. You can check by running the following command in your terminal:

git --version

If Git is not installed, visit Git Downloads to install it.

2. Initialize a New Git Repository

Navigate to the folder where you want to create a new project. Use the following command to initialize a Git repository:

git init "your-project-name"

This creates a new repository in the folder. Navigate into it using:

cd your-project-name

3. Create Files

Create the files you want to track. For example, create an HTML file using:

touch index.html

4. Check the Status of Your Repository

To check the status of your repository and see untracked changes (files Git is not tracking), use:

git status

5. Stage Changes for Tracking

Stage your changes so Git can start tracking them:

git add index.html

6. Commit Changes

Once the changes are staged, commit them with a message describing the changes:

git commit -m "Create index.html"

This creates a snapshot of the current state of your project.

7. Create More Files and Track Changes

You can now create additional files (e.g., a CSS file) and follow the same process to track and commit changes:

touch style.css
git add style.css
git commit -m "Create style.css"

8. Create and Switch to a New Branch

Branches allow you to work on different versions of your project without affecting the main branch. Create a new branch:

git branch feature-branch

Switch to the new branch:

git checkout feature-branch

9. Link Your Local Git with a GitHub Repository

Now, you can connect your local Git repository to a remote repository on GitHub to back up your code or collaborate with others.

Steps to Link Your Repository:

  1. Go to GitHub and sign in or create a new account.
  2. Create a new repository by clicking on the **New** button in the top-right corner.
  3. Give the repository a name and click **Create repository**.
  4. In your terminal, add the remote link to your local repository: 
    git remote add origin https://github.com/your-username/your-repo-name.git
  5. Push your changes to the remote repository: 
    git push -u origin master
  6. GitHub will now have a copy of your code in the repository, and you can collaborate with others or track changes remotely.

10. Push and Pull Changes

Once your repository is linked with GitHub, you can push updates to it or pull changes from others.

Push changes using:

git push origin master

Pull the latest changes from GitHub (useful when working with others):

git pull origin master

By following these steps, you've set up a Git repository, tracked changes, and linked it with a GitHub remote repository. Keep practicing to get comfortable with Git commands!
